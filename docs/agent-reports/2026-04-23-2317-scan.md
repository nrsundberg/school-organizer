### [P0] Signup step 1 â†’ step 2 transition reverts to step 1 (critical path broken)

- **Where:** `/signup?plan=car-line` â€” clicking Continue after filling step 1 does not advance the user to step 2.
- **What:** The `POST /api/auth/sign-up/email` request returns `200 OK` (the account is created server-side), the URL briefly updates to `?plan=car-line&step=2`, `/signup.data?plan=car-line&step=2` loads (200) â€” then the UI immediately navigates back to `?plan=car-line&step=1` and re-fetches `/signup.data?plan=car-line&step=1` (200). The "Your school" heading/slug input is never rendered. No inline error text is shown to the user. For the end user the effect is: "I hit Continue and nothing happened."
- **Evidence:**
  - API calls observed (`flow-tmp.mjs`):
    ```json
    [
      { "url": ".../api/auth/sign-up/email", "status": 200, "method": "POST" },
      { "url": ".../signup.data?plan=car-line&step=2", "status": 200, "method": "GET" },
      { "url": ".../signup.data?plan=car-line&step=1", "status": 200, "method": "GET" }
    ]
    ```
  - URL at timeout: `/signup?plan=car-line&step=1`
  - `result.errorText: []` â€” no inline error rendered.
  - `result.pageErrors: []`, `result.consoleErrors: []` â€” no thrown errors, so this is a state-machine bug, not a runtime crash.
  - Visible body text at timeout: still the step-1 form ("Create your account / Your name / Email / Phone number / Password / Confirm password / Continue").
- **Reproduction (Playwright):**
  ```ts
  const email = `scanner+${Date.now()}@test.pickuproster.com`;
  await page.goto("/signup?plan=car-line");
  await page.locator('input[autocomplete="name"]').first().fill("Scanner Test");
  await page.locator('input[type="email"]').first().fill(email);
  await page.locator('input[type="tel"]').first().fill("5551234567");
  const pw = page.locator('input[type="password"]');
  await pw.nth(0).fill("password1234");
  await pw.nth(1).fill("password1234");
  await page.getByRole("button", { name: /^Continue$/ }).click();
  // Expected: step 2 "Your school" heading + slug input
  await page.waitForSelector('h1:has-text("Your school"), input[name="slug"]');
  // Actual: times out; URL returns to ?step=1 after briefly flashing ?step=2
  ```
- **Suspected cause:** `app/routes/auth/signup.tsx` (lines ~229-233) has
  ```ts
  useEffect(() => {
    if ((step === 2 || step === 3) && !isAuthed) {
      updateStepParam(1, { replace: true });
    }
  }, [step, isAuthed, updateStepParam]);
  ```
  `isAuthed` is derived from the root loader's `user` via `useRouteLoaderData("root")`. After `signUp.email()` succeeds and sets the session cookie, `setStep(2)` is called which triggers a React Router navigation. React Router revalidates the **signup** route's loader (we see `/signup.data?step=2` fetched) but the **root** loader is not being revalidated â€” so `rootData.user` is still `null` when the "bounce back to step 1" `useEffect` runs. This appears to be a regression from the better-auth wiring, or a `shouldRevalidate` config on the root route that excludes signup navigation. Fix is likely either to call `revalidator.revalidate()` in `handleStep1` immediately after `signUp.email()` resolves, or to set local `justSignedUp` state in the component and short-circuit the bounce-back effect. The 2026-04-23-1544-scan report claimed step-2 was "reached (password input visible)" â€” but password inputs only exist on step 1 (step 2 has name/slug inputs), so that earlier claim was a false positive; this bug was already present then and went undetected.
- **Status:** attempted: staging-deploy (branch `agent-fix/2026-04-24-signup-step-bounce-back` @ `f7a1f02` â€” fix committed, typecheck + 102 unit tests pass, but `wrangler d1 migrations apply --env staging` fails with `CLOUDFLARE_API_TOKEN` missing, blocking the staging-smoke gate. Same blocker as the 2026-04-24 nightly build. Branch pushed for human review. Fix: `useRevalidator().revalidate()` after `signUp.email()` resolves + local `justSignedUp` flag short-circuits the bounce-back `useEffect` during the loader-refresh race. Regression spec: `e2e/flows/signup-step-bounce.spec.ts`.)
