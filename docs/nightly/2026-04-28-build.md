## What landed

Two new admin routes plus their server-side helpers, unit tests, and
i18n + sidebar surface area:

- **`/admin/data-export`** (`app/routes/admin/data-export.tsx`, ~340 LOC)
  - Loader: `protectToAdminAndGetPermissions` + plan gate via
    `planAllowsReports(org.billingPlan)`. Returns row-count preview from
    parallel `count()` calls and a "last exported at" derived from the
    most-recent `OrgAuditLog` row with `action = "data.export"`.
  - Action: rebuilds the gate (so a crafted POST from a downgraded org
    doesn't slip through), paginates `findMany` with `take: 5000` per
    table, applies the column whitelist (see below), assembles
    `manifest.json` + per-table `*.json` entries via `fflate.zip`, and
    streams back as `application/zip` with `Content-Disposition:
    attachment`. Audit-log write happens after a successful encode
    (best-effort — a failure here doesn't block the download).
  - Component: row-count preview card + download button + "Danger zone"
    footer linking to `/admin/data-delete`.

- **`/admin/data-delete`** (`app/routes/admin/data-delete.tsx`, ~280 LOC)
  - Loader: row-count snapshot + `hasRecentExport` flag.
    **Intentionally not plan-gated** per the queue scope — a downgraded
    org needs to be able to remove its data regardless of subscription
    tier.
  - Action: validates the typed-slug + acknowledgement checkbox, snapshots
    `rowCountsBefore` for the audit payload, then walks the FK chain in
    a single `prisma.$transaction`. Order respects dependents → parents
    (`programCancellation` → `afterSchoolProgram`, `callEvent` → `space`,
    etc.). The `Org` row itself plus every Stripe-linked column on it is
    preserved; the executing admin's `User` row is also preserved
    (Open Q #2 / Option A) so the action doesn't kill its own session.
    Redirect to `/admin?dataDeleted=1` on success.
  - Component: red-themed warning card, slug-typed-confirmation input
    (must match exactly), "irreversible" checkbox, danger-styled submit.

- **`app/lib/data-export.server.ts`** (~155 LOC) — pure helpers usable
  from both the route and the test suite:
  - `EXPORT_WHITELIST` — per-table allowlist of column names. **Positive
    list**: every shipped column has to be opted in by name, so future
    schema additions default to NOT being exported. The `users`
    whitelist deliberately omits `passwordHash` / token / 2FA fields;
    `appSettings` omits `viewerPinHash`.
  - `whitelistRow` — strips a row to whitelisted keys + normalizes
    `Date` instances to ISO strings.
  - `buildManifest` — top-level `manifest.json` shape with
    `schemaVersion: "1"` so a future format change can be detected.
  - `DELETE_ORDER` — the canonical FK-respecting delete order. Lives
    here so the route's transaction body and the unit test's snapshot
    can both reference the same array literal.

- **`app/lib/data-export.server.test.ts`** (~140 LOC) — 7 unit tests:
  - `whitelistRow` keeps only whitelisted columns.
  - `whitelistRow` normalizes `Date` → ISO string.
  - **Hardened regression test**: even when given a row containing
    `passwordHash` / `sessionToken` / `twoFactorSecret` / `stripeCustomerId`,
    the user whitelist drops all four. (This is the test that prevents
    the highest-stakes future regression: a schema column added without
    a security review accidentally leaking into `users.json`.)
  - `appSettings` whitelist drops `viewerPinHash`.
  - `buildManifest` shape check.
  - `EXPORT_WHITELIST` covers every `ExportTable` (compile-time + runtime).
  - `DELETE_ORDER` puts `user` last and `dependent → parent` everywhere
    that matters.

- **`app/routes.ts`** — register both new routes under the `admin/*` prefix.
- **`app/components/admin/AdminSidebar.tsx`** — `Download` icon + new

### Off-script workarounds

The repo's `npm run typecheck` script runs `react-router typegen && tsc`,
and `react-router typegen` calls `unlink` on `.react-router/types/+future.ts`
(among others) — the fuse mount blocks this. Two workarounds applied,
both equivalent to a clean typegen run on a non-fuse checkout:

1. **`+routes.ts` patched in place.** Added `Pages` / `RouteFiles` /
   `RouteModules` / page-union entries for the two new routes. The
   patch is small and the file is gitignored, so this is a worktree-only
   convenience for the typecheck — a fresh typegen run on the next
   non-fuse checkout will produce the same content.
2. **`+types/data-export.ts`, `+types/data-delete.ts` written manually.**
   Mirror of the `branding.ts` shape. Same caveat: gitignored, will
   regenerate cleanly elsewhere.

While I was patching `+routes.ts`, two other entries needed manual
addition because they were missing on master too: `routes/admin/drills.history.tsx`
and `routes/admin/drills.history.$runId.tsx`. Those route files landed
in commit `3580ca6b` but their `+types` were never regenerated and
committed; that's why the 2026-04-26-polish "typecheck clean on
