### Fuse-sandbox git note

`git merge` kept failing with `Unable to create '.git/index.lock':
File exists` every attempt — every merge run left its own
`index.lock` / `ORIG_HEAD.lock` behind because the fuse mount blocks
`unlink()`, and every subsequent run refused to proceed. Standard
`mv`-away-the-lock cleanup (per AGENTS.md) worked once, then the
next `git` invocation re-created its own un-unlinkable lock and the
cycle repeated. Six retries later, the merge landed via plumbing
(`git read-tree` into a `GIT_INDEX_FILE=$(mktemp)` temp index →
`git write-tree` → `git commit-tree` with both parents →
`git update-ref refs/heads/master`), which bypasses the main index
entirely. Recommending AGENTS.md adds a troubleshooting note so the
next agent that hits the stuck-index-lock loop reaches for plumbing
instead of retrying `git merge` indefinitely.
