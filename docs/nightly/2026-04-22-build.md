### `.github/workflows/e2e.yml` — three-shard matrix

The existing workflow installed `chromium` only and ran `npm run test:e2e`,
which since 0b runs all three Playwright projects. The iPhone 13 device
emulation needs WebKit, which wasn't being installed, so the mobile-iphone
project would have failed on a fresh CI runner. New shape:

- Matrix over `shard` with three entries:
  - `desktop-chromium` → project `chromium`, browser `chromium`
  - `mobile-iphone`    → project `mobile-iphone`, browser `webkit`
  - `mobile-pixel`     → project `mobile-pixel`, browser `chromium`
- `fail-fast: false` — mobile and desktop regressions surface together.
- `timeout-minutes: 15` per shard (enforces the queue-spec budget).
- Cache key scoped by browser (`playwright-<browser>-<lockhash>`) so
  shards don't invalidate each other. The two `chromium` shards share a
  cache hit; the webkit shard has its own.
- Per-shard report artifact on failure
  (`playwright-report-<shard-name>`).
- `npx playwright test --project=${{ matrix.shard.project }}` scopes
  each shard to one project.
