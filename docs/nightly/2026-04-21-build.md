## Bugs found by the sweep (real — require a follow-up workstream)

Both are kept as `test.fixme(...)` in `e2e/smoke.spec.ts` so CI stays green
while the bugs are visibly tracked in the Playwright report as "expected to
fail". Remove the fixmes when the fix lands.

### BUG 1: `/api/healthz` redirects to `/login` for unauthenticated requests

- Route: `app/routes/api/healthz.ts`
- Root cause: `app/domain/utils/global-context.server.ts` middleware.
  `anonSkipsViewer` whitelists `/api/auth/*`, `/api/check-email`,
  `/api/check-org-slug`, `/api/branding/logo/*`, but not `/api/healthz`.
  When an unauthenticated, no-session, no-org request hits the marketing host
  (`localhost`, apex, or `www`), the code at line ~190-192 throws
  `redirect("/login?next=" + encodeURIComponent(pathname))`.
- Impact: external uptime monitors (UptimeRobot, Cronitor, etc.) will see a
  `302 Location: /login?next=/api/healthz` and count that as "up" only if
  they follow redirects — which then hits the HTML login page. The whole
  point of `/healthz` is an unauthenticated JSON endpoint.
- Fix: add `/api/healthz` to `anonSkipsViewer` (or, cleaner, make
  `isApi && pathname === "/api/healthz"` short-circuit to `next()`).

### BUG 2: `/status` redirects to `/login` on marketing host

- Route: `app/routes/status.tsx` (loader calls `getStatusPageData(context)`).
- Root cause: Same middleware as above. `publicMarketingPath` lists
  `/pricing`, `/faqs`, `/signup`, `/`, and a couple of onboarding APIs, but
  not `/status`. The page has a caching header and is obviously intended to
  be public, but the global middleware kicks unauth visitors to login first.
- Impact: the `/status` page linked from the marketing footer
  (`<a href="/faqs#status">`) and anywhere else is broken for logged-out
  visitors, which is the entire target audience of a status page.
- Fix: add `pathname === "/status"` to `publicMarketingPath`.

Both bugs are one-line fixes in `app/domain/utils/global-context.server.ts`.
Recommend a follow-up workstream `0a-fix` (or rolled into a small PR) rather
than bundling them into this test-only branch — the fixmes in the test file
will flip to passing once the middleware ships.
