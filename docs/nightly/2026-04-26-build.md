## Implementation note — fixture option scope

The polish agent's research spec recommended `{ option: true }`
(test-scoped). Playwright's typing requires options to be
**worker-scoped** (`{ option: true, scope: "worker" }`); a test-scoped
option triggers `TS2741: Property 'scope' is missing`. Worker scope
still supports `test.use({ tenantBillingPlan: ... })` inside a
`describe` block — Playwright simply allocates a new worker for each
distinct value in a single test run. The `tenant` fixture is per-test
and consumes the worker-scoped option (the consumer-scope rule allows
test-scoped fixtures to consume worker-scoped ones). Net effect:
identical spec ergonomics to the proposal, plus the typing is sound.
